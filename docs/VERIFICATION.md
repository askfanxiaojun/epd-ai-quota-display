# 实际验证记录

本项目不是只完成了代码编写，以下链路均在真实 Mac 和电子价签上验证通过。

## 硬件与连接

- 设备广播名：`NRF_EPD_0819`；
- 芯片：nRF52811；
- 屏幕：400×300、SSD1619、黑白红三色；
- 协商后的特征值长度：`mtu=244`；
- Mac 可以通过 Bleak 扫描、连接和接收设备通知。

## 显示验证

- 固件内置日历可以正常显示；
- 固定测试卡可以正常显示；
- Codex 真实配额页面可以正常显示；
- 黑色图层完整发送：`62/62 chunks`；
- 红色图层完整发送：`62/62 chunks`；
- `REFRESH` 命令发送成功；
- 人工使用 `--clear-first` 时，清屏后会再次初始化；自动更新默认不预清屏。

## 数据验证

- 可以从本机 Codex ChatGPT OAuth 会话读取配额；
- 可以解析 5 小时和 7 天两个窗口；
- 剩余比例按 `100 - used_percent` 计算；
- 重置时间可以转换为本地日期和时间；
- OAuth token 不会输出到日志或画面。

## 版式边界验证

- 服务名称、窗口标题、百分号、重置时间和更新时间已经针对实屏放大；
- 使用两个 `99%` 窗口验证大数字与进度条不会重叠；
- 使用真实数据 `88% / 74%` 再次写入实体屏；
- 该次写入的黑色和红色图层均为 `62/62`，任务退出码为 `0`。

## 定时任务验证

- 用户级 LaunchAgent 注册成功；
- 登录后可以自动加载；
- 每 1800 秒运行一次；
- 后台环境可以访问网络和蓝牙；
- 可以使用 `launchctl kickstart` 立即触发；
- 成功任务退出码为 `0`；
- 自动更新直接覆盖完整图层，扫描或传输失败时不会先把实体屏幕清成白色；
- BLE 中途断开时会重新扫描并重试一次完整图层；

需要注意：BLE 广播并非每次扫描都一定命中。一次手动触发中曾出现扫描不到设备，
第二次触发即正常发现并完成写入。当前版本已经加入一次有限重试，并将写入确认
间隔缩短为每 20 个分块一次。

## 日历环境样式验证

新增的 `calendar-sensor` 模式已完成以下本地验证：

- 400×300 左月历、右温湿度页面可以正常生成；
- 黑色层和红色层均为 15,000 字节；
- 当天日期可以随系统日期变化并使用红圈标记；
- 温度和湿度 JSON 字段支持嵌套键；
- 湿度缺失时页面不会显示湿度模块；
- 超过设定时间的旧传感器数据会在连接墨水屏前终止；
- JSON 文件、命令行实测值和演示数据路径均通过测试；
- LaunchAgent 模板支持配置 15 分钟等不同刷新间隔。

日历环境样式尚未绑定某个具体品牌的实体温湿度传感器，也没有用演示数据覆盖
真实墨水屏。接入实际传感器后，应再补充一次“传感器 → Mac → BLE → 实屏”的
端到端验证记录。
